How We Built It
The Engineering Behind DealCart
INTELLIGENT SCRAPING

We use a dual-strategy scraping engine — first attempting structured JSON-LD extraction (fastest, most reliable), then falling back to HTML selectors with Cheerio. Built-in retry logic with exponential backoff ensures resilience against rate-limiting.

FAKE SALE ALGORITHM

Our V-curve detection algorithm analyzes the last 10 price data points, identifies artificial hikes (≥15%), quantifies the subsequent "discount", and flags products where the current price remains near or above the pre-hike baseline.

DEAL SCORING ENGINE

A multi-factor scoring system (0-100) evaluates each product against its all-time low, average price, recent trends, and volatility to generate actionable buy/wait recommendations.

ROW LEVEL SECURITY

Every database table is protected by Supabase RLS policies. Users can only view, insert, update, or delete their own data — ensuring complete multi-tenant isolation with zero middleware overhead.

Fake Sale Detection Algorithm
V-Curve Pattern Analysis
Detection Criteria
  • 1. Minimum 5 data points required
  • 2. Price hike ≥ 15% from baseline
  • 3. Subsequent drop ≥ 10% from peak
  • 4. Current price still near baseline
  • 5. Pattern must form V-curve shape
Result: Products matching this pattern are flagged with a FAKE SALE badge on the dashboard.

Scalability & Performance
BUILT FOR HIGH WORKLOADS

DealCart is architected to scale seamlessly as user count grows and the tracking volume increases. Using cloud-native tooling and decoupled asynchronous processing, it maintains high performance even under heavy loads.

Edge Network Caching
By leveraging Next.js on Vercel, static assets and dynamic routes utilize intelligent CDN caching. Users get single-digit millisecond latency regardless of geographic location.
Connection Pooling
To prevent database overload during traffic spikes, Supabase utilizes PgBouncer for efficient PostgreSQL connection pooling, allowing thousands of concurrent queries without server strain.
Decoupled Cron Jobs
Price checking operations run asynchronously via Vercel Cron. Large batches of products are processed in chunks over serverless functions, preventing UI blocking and timeout errors.
Exponential Backoff
The scraper engine includes resilient error parsing and retry constraints. If Amazon/Flipkart imposes momentary rate limits, DealCart backs off exponentially to prevent permanent IP blocks.
